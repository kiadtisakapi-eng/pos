1 นำแอปขึ้นเว็บ

iPad เปิดไฟล์จากคอมโดยตรงไม่ได้ และระบบ PWA (ติดตั้ง+ออฟไลน์) ต้องการ HTTPS จึงต้องนำไฟล์ขึ้นเว็บก่อน เลือกวิธีใดวิธีหนึ่ง

วิธี A — Netlify (ง่ายสุด ไม่ต้องมีบัญชี Git) แนะนำสำหรับเริ่มต้น

  1. บนคอม เปิด https://app.netlify.com/drop
  2. ลาก ทั้งโฟลเดอร์ โปรเจค POS ไปวางในหน้าเว็บ
  3. ระบบจะอัปโหลดและให้ลิงก์ HTTPS ทันที เช่น https://ชื่อสุ่ม.netlify.app
  4. (ทางเลือก) สมัครบัญชีฟรีเพื่อเก็บลิงก์ไว้ถาวรและตั้งชื่อเองได้

วิธี B — GitHub Pages (ลิงก์ถาวร เหมาะใช้จริงระยะยาว)

  1. สมัคร/ล็อกอิน github.com → กด New repository ตั้งชื่อ เช่น erotica-pos → Create
  2. ในหน้า repo กด Add file → Upload files แล้วลากไฟล์ ทั้งหมด ในโฟลเดอร์นี้ขึ้นไป (รวม .nojekyll ที่ผมเตรียมไว้) → Commit changes
  3. ไปที่ Settings → Pages
  4. หัวข้อ Source เลือก Deploy from a branch → Branch = main โฟลเดอร์ / (root) → Save
  5. รอ 1–2 นาที จะได้ลิงก์ https://<ชื่อผู้ใช้>.github.io/<ชื่อ repo>/
หมายเหตุ: ไฟล์ .nojekyll (ไฟล์ว่าง) สำคัญ — กันไม่ให้ GitHub ประมวลผลแบบ Jekyll อย่าลบทิ้ง

2 ติดตั้งบน iPad (เพิ่มลงหน้าจอ)

  1. บน iPad เปิดลิงก์ที่ได้จากขั้นตอนที่ 1 ด้วย Safari
    สำคัญ: ต้องเป็น Safari เท่านั้น — Chrome บน iPad ติดตั้ง PWA แบบเต็มจอ/ออฟไลน์ไม่ได้
  2. แตะปุ่ม แชร์ ⬆️ สี่เหลี่ยมมีลูกศรขึ้น ที่แถบด้านบน/ล่าง
  3. เลื่อนหาเมนู "เพิ่มไปยังหน้าจอโฮม" (Add to Home Screen) → แตะ
  4. ตั้งชื่อแอป (เช่น "POS ร้าน") → แตะ เพิ่ม
  5. จะมีไอคอนแอปขึ้นบนหน้าจอ iPad — เปิดจากไอคอนนี้จะได้แบบ เต็มจอเหมือนแอปจริง
ครั้งแรกต้องต่ออินเทอร์เน็ต เพื่อโหลดและแคชไฟล์ หลังจากนั้นเปิดใช้แบบออฟไลน์ได้

3 Deploy Backend (Google Apps Script)

ส่วนนี้ทำให้ระบบ sync ยอดขาย/สรุปขึ้น Google Sheets และสำรองข้อมูลขึ้น Google Drive อัตโนมัติ

  1. เปิด Google Sheets สร้างไฟล์ใหม่ 1 ไฟล์ (จะใช้เก็บข้อมูลทั้งหมด)
  2. เมนู Extensions → Apps Script
  3. ลบโค้ดตัวอย่างที่มี แล้ว วางโค้ดทั้งหมด จากไฟล์ google_apps_script.js → กด บันทึก (รูปแผ่นดิสก์)
  4. กด Deploy → New deployment
  5. กดรูปเฟือง ⚙️ ข้าง "Select type" → เลือก Web app
  6. ตั้งค่าให้ตรงนี้ เป๊ะ ๆ:
    ช่องตั้งเป็น
    Execute asMe (อีเมลคุณ)
    Who has accessAnyone
  7. กด Deploy → อนุญาตสิทธิ์ (Authorize) → เลือกบัญชี → "Advanced" → "Go to (project) (unsafe)" → Allow ทำครั้งเดียว
  8. คัดลอก Web app URL (ลงท้าย /exec) เก็บไว้ใช้ในขั้นตอนที่ 4
เมื่อแก้โค้ด GAS ภายหลัง: ต้องไป Deploy → Manage deployments → ✏️ Edit → Version: New version → Deploy ทุกครั้ง ไม่งั้นของเก่าจะยังทำงานอยู่ (URL เดิมใช้ได้ต่อ)
โครงสร้างชีตที่ระบบจะสร้างให้อัตโนมัติ: สรุปรายเดือน (master), MM-yyyy (รายการบิล), สรุป-MM-yyyy และ สรุป-YYYY-MM-DD — ไม่ต้องสร้างเอง
